# FIRST LIGHT: TERRA - 제출용 수정 기획서

- 기준일: 2026-07-22
- 제출 버전: Build Week P0 local prototype
- 트랙: Apps for Your Life
- 핵심 대상: 작은 행동을 시작하고 사람과 따뜻하게 연결되고 싶은 개인
- 서버 상태: 미구현. 인증·동기화·저장·검증은 다음 단계 계획
- 영상 기준: 16:9, 약 2분, 제품 데모 중심

## 1. 한 문장 기획

FIRST LIGHT: TERRA는 혼자 시작한 작은 행동과 따뜻한 신호가 TERRA의 공동기지와 다음 항로를 조금씩 복구해 나가는 소셜 우주 탐사 게임입니다.

## 2. 처음 기획에서 바뀐 점

초기 기획은 Firebase 기반 실시간 멀티플레이, 전체 채팅, presence, 사진 탐사 기록, 공동 건설 서버 검증까지 포함한 온라인 소셜 우주를 한 번에 보여주는 방향이었습니다. 현재 제출 버전은 서버가 연결된 것처럼 표현하지 않고, 구현과 검증이 끝난 사용자 경험 한 줄을 로컬 수직 슬라이스로 보여주는 방향으로 범위를 좁힙니다. 서버 기능은 아래의 다음 단계 계획으로만 설명합니다.

| 항목 | 초기 구상 | 현재 제출 기준 |
| --- | --- | --- |
| 세계의 중심 | 온라인 공동 우주 전체 | TERRA 숲 탐험과 공동기지 복구의 한 루프 |
| 진입 경험 | 로그인 후 여러 온라인 기능으로 진입 | 시작 화면 -> 로컬 프로토타입 계정 흐름 -> 캐릭터 생성 -> 항로 관제실 |
| 플레이 공간 | 2D 소셜 공간과 여러 메뉴를 폭넓게 시연 | 수채화풍 3D TERRA 숲에서 이동, 자원 수집, 상호작용 |
| 공동체 기능 | Firebase 실시간 presence와 서버 동기화 | 브라우저 메모리 기반 교신, 시그널, 프로젝트 진행 UI |
| 공동 행동 | 실제 행동 기록과 사진 공유를 서비스로 연결 | 메뉴와 로컬 미션·신호 보관소의 경험을 프로토타입으로 제시 |
| 캐릭터 | 독립 바디·헤드·의상 조합 | 품질 검수된 남성/은발 여성 고정 프리셋과 4방향 모션 |

## 3. 현재 데모가 증명하는 것

### 문제

온라인 공간에서는 말만 남거나 쉽게 소비되고, 오프라인에서는 작은 행동을 시작해도 그것이 다른 사람과 연결되는 경험이 약합니다.

### 제안

플레이어가 TERRA에 들어와 직접 움직이고 자원을 모은 뒤, 그 결과를 공동기지와 LUNA 항로의 진행도로 확인하게 합니다. 행동의 보상은 경쟁 점수가 아니라 다음 항로를 함께 여는 감각입니다.

### 데모의 핵심 증거

1. 캐릭터를 만들고 자신의 탐사대원으로 진입한다.
2. 항로 관제실에서 TERRA는 입장 가능하고 LUNA는 공동 복구 중임을 확인한다.
3. `바람이 머무는 숲`에서 WASD 또는 방향키로 이동하고 나무·바위에 `F`로 상호작용한다.
4. 수집한 자원이 상단 HUD와 프로젝트의 개인 보유 자원에 반영된다.
5. 프로젝트에서 자원을 공동 창고에 넣으면 TERRA 공동기지 복구율과 LUNA 진행도를 확인한다.

## 4. 제출 영상에서 전달할 한 가지 메시지

**작은 행동 하나가 내 인벤토리에만 머무르지 않고, 모두의 다음 항로를 여는 기록이 된다.**

영상에서는 시작 화면, 캐릭터 생성, 항로, 숲 탐험, 자원 수집, 프로젝트 기여를 한 흐름으로 연결합니다. 광장·시그널·미션 UI는 보조 컷으로만 사용하고, 핵심 루프를 흐릴 정도로 기능을 나열하지 않습니다.

## 5. 기능 범위와 표현 기준

| 기능 | 제출 영상 표현 | 설명할 때의 정확한 표현 |
| --- | --- | --- |
| 시작/캐릭터 생성 | 필수 | 로컬 개발 모드의 프로필 생성 흐름 |

| Firebase/실시간 멀티플레이 | 제외 | 아직 검증 전인 다음 단계, 완료 기능처럼 말하지 않음 |

## 6. AI 활용 설명

AI는 플레이어를 대신해 신호를 작성하는 런타임 캐릭터가 아니라, 제작 속도와 반복 검수를 높이는 개발 도구로 사용했습니다.

- 전체 개발 작업의 약 99%: Codex 협업으로 진행
- Codex와 GPT-5.6: 화면 흐름, 상태 전환, 렌더링 구조, 캐릭터 모션, 테스트와 문서화
- 이미지 생성 워크플로우: TERRA 배경과 환경 콘셉트 에셋
- `generate2dsprite`: 남성/은발 여성 캐릭터의 4방향 걷기·대기·달리기·상호작용 시트
- 소스 테스트와 빌드: 구현 범위와 회귀 여부를 빠르게 확인

최종 방향 선택과 실행 검증은 프로젝트 제작자가 담당했습니다. Codex가 작성하거나 수정한 결과도 테스트와 실제 실행 흐름을 통해 확인했습니다.

세계관 안에서 AI는 인간의 신호를 대신 만드는 주체가 아니라, 기록과 항로를 보존하는 장치로 정의합니다. 공동체를 만드는 것은 플레이어의 행동입니다.

## 7. 서버 구현 계획

현재 저장소에는 배포 가능한 서버 백엔드가 없으며, 프로필·채팅·인벤토리·공동 진행도는 브라우저 메모리와 로컬 상태로 동작합니다. 다음 서버 구현은 아래 순서로 계획합니다.

1. Firebase Authentication과 Firestore 프로필 저장
2. Realtime Database 기반 presence, 전체 채팅, 근거리 말풍선
3. Firestore 기반 공동 진행도, 미션 이벤트, 신호 기록
4. 서버 검증 기반 자원 수집·공동 건설 API
5. Cloud Storage 기반 사진 탐사 기록과 갤러리
6. 권한·오프라인 재접속·동시성 충돌 처리 검증

이 목록은 아키텍처와 다음 작업 순서를 설명하기 위한 계획이며, 이번 제출 버전에 구현되었거나 연결된 서버 기능을 의미하지 않습니다.

## 8. 이번 제출에서 약속하지 않는 것

- Firebase Authentication, Firestore, Realtime Database, Cloud Storage가 연결된 상태라고 말하지 않습니다.
- 여러 브라우저의 presence나 실시간 채팅 동기화를 완료했다고 말하지 않습니다.
- 로컬 개발 모드의 프로필, 채팅, 진행도를 영구 클라우드 저장으로 표현하지 않습니다.
- 사진 탐사 기록과 미션 보관소를 실제 서비스 데이터로 표현하지 않습니다.
- 서버 검증, 권한 처리, 동시성 충돌 방지가 완료되었다고 말하지 않습니다.
