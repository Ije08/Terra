# Quill 기록

- 개발 운영 문서는 루트 `AGENTS.md`를 기준으로 두고, `README.md`와 `docs/build-week/STATUS.md`에는 같은 규칙을 프로젝트 맥락에 맞게 요약한다.
- 프론트 검증은 사용자 제보 또는 명시적인 브라우저 검증 요청이 있을 때만 수행한다. 일상적인 변경은 소스 테스트·관련 린트·빌드로 확인해 피드백 사이클을 짧게 유지한다.
- 이번 문서화는 실행 규칙을 새로 만들고 Build Week에 AI 작업 순서와 시간 단축 근거를 기록하는 범위로 제한했으며 런타임 코드는 변경하지 않았다.
