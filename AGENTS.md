# TERRA 개발 에이전트 운영 규칙

## 프론트 검증 기준

- 사용자가 브라우저에서 즉시 확인할 수 있는 프론트 변경은 기본 작업 범위에 브라우저 검증을 포함하지 않는다.
- 기본 검증은 변경 범위에 맞는 소스 테스트, 타입/린트, 빌드처럼 빠르게 재현 가능한 검사로 우선한다.
- 사용자가 “기능이 안 된다”, “확인해봐”, “브라우저에서 재현해봐”처럼 동작 검증을 요청한 경우에만 브라우저 검증을 수행한다.
- 브라우저 검증이 필요한 경우에도 전체 플로우를 반복하지 말고, 사용자가 제보한 기능과 관련된 최소 경로만 확인한다.
- 검증하지 않은 항목은 검증 완료처럼 보고하지 않는다. 실행한 검사와 생략한 검사를 최종 보고에 구분한다.

## 작업 속도 원칙

- README와 `docs/build-week/STATUS.md`를 먼저 읽고 기존 결정과 현재 상태를 재사용한다.
- 이미지 에셋, 프론트 구현, 문서화처럼 독립적인 작업은 가능한 경우 세션을 분리해 병렬 진행한다.
- 관련 파일만 읽고 수정하며, 전체 저장소 검사나 브라우저 자동화는 사용자 요청 또는 기능 불량 재현에 필요할 때만 실행한다.
- 작은 변경은 한 번에 구현하고, 같은 변경 범위에 대한 테스트와 빌드를 묶어 실행한다.
